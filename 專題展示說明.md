# 售票資訊整合 App 展示說明

## 專題定位

本專題是「售票資訊整合」手機 App，將不同售票來源的活動資料整合到同一個 App，提供活動列表、關鍵字搜尋、搶票提醒與資料分析。

## 已整合功能

1. **活動列表**
   - 從 MySQL 資料庫讀取售票活動。
   - 顯示活動名稱、藝人、活動時間、搶票時間、場地、票價、來源網站。

2. **搜尋功能**
   - 可輸入藝人、活動名稱、場地、票價等關鍵字。
   - App 會呼叫後端 API 即時查詢資料庫。

3. **搶票提醒**
   - 可從活動卡片加入提醒。
   - 後端會把提醒存進 MySQL 的 `reminders` 資料表。
   - 提醒頁面可查看與刪除提醒。

4. **資料分析**
   - 票價區間分析。
   - 活動月份統計。
   - 熱門場地排行。
   - 整合原本 `price.py`、`activity_time.py`、`venue.py` 的分析概念，改成 App 可直接讀取的 API。

## 系統架構

```text
Android 手機 App
        ↓
Node.js Express API
        ↓
MySQL / Aiven Cloud Database
```

## 給老師看的展示流程

1. **先介紹題目**
   - 題目：售票資訊整合
   - 目的：把分散在不同售票平台的活動資料集中到同一個手機 App。

2. **展示首頁**
   - 顯示總活動數、藝人數、場地數、提醒數。
   - 展示最新售票活動列表。

3. **展示搜尋**
   - 搜尋一個藝人或場地，例如「台北」、「Zepp」、「演唱會」。
   - 說明這是從 MySQL 即時查詢，不是寫死資料。

4. **展示提醒**
   - 從活動卡片點「加入提醒」。
   - 切到提醒頁面確認資料已新增。
   - 可刪除提醒。

5. **展示分析**
   - 顯示票價區間。
   - 顯示熱門月份。
   - 顯示熱門場地。

6. **最後說明架構**
   - 手機 App 負責畫面與操作。
   - Node.js 後端負責 API。
   - MySQL 負責儲存售票資料與提醒。

## 實體手機展示注意

目前 App 預設給 Android 模擬器使用：

```text
http://10.0.2.2:4000
```

如果要用實體手機展示，需要把 `mobile-app/app/src/main/java/com/example/myapplication/MainActivity.kt` 裡的 `API_BASE_URL` 改成電腦的區網 IP，例如：

```text
http://192.168.1.23:4000
```

手機和電腦必須連同一個 Wi-Fi。
